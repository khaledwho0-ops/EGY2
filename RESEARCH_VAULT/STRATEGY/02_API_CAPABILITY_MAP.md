## 0. AI Provider & Rotator Infrastructure (read this first)

EAL has **four overlapping AI abstraction layers**. Knowing which one a route uses tells you its failover behavior.

| Layer | File | What it is | Cascade order |
|---|---|---|---|
| **NVIDIA-First helper** | `src/lib/ai/nvidia-first.ts` | The most-used helper. `nvidiaFirstGenerate()` / `nvidiaFirstGenerateJSON()`. Raw `fetch` to NVIDIA NIM, 22s timeout, then Gemini SDK fallback. | NVIDIA NIM (Nemotron-3 Ultra 550B) → Gemini 2.0 Flash |
| **Mega-Rotator v7** | `src/lib/debunking/gemini-rotator.ts` | Structured-output (`generateObject`) engine. Up to ~13 key slots across 6 providers with cooldown/round-robin. Also exposes streaming model getters. | Gemini ×5 → Groq ×4 → NVIDIA ×5 → OpenRouter ×2 → Cerebras ×2 → Together → SambaNova (Kimi disabled) |
| **Multi-provider engine** | `src/lib/ai/providers.ts` | `aiGenerate()` plain-text cascade + `analyzeClaimWithAI`, `mentalHealthAI`, `analyzeSentiment`, `translateText`. | NVIDIA NIM → NVIDIA DeepSeek V4 Flash → Gemini → Groq → GitHub Models (gpt-4o-mini) → HuggingFace (Mistral-7B) |
| **AI-SDK router** | `src/lib/ai/router.ts` | Task→model map used by RAG/embeddings. | verify=gpt-4o, fast_classify=Groq llama-3.3, embed=OpenAI text-embedding-3-small, ar_tafsir=Gemini, safety=gpt-4o-mini |
| **LiteLLM-style** | `src/lib/ai/llm-provider.ts` | Client-side provider switch w/ static fallback (Claude/OpenAI/Llama). Largely legacy; references `/api/llm/*` routes that are not present. | claude-sonnet-4 → gpt-4o → llama-3.1-70b → static |

**Supporting AI services**
- **Cohere Rerank** (`src/lib/ai/cohere-rerank.ts`) — `rerank-v3.5` multilingual (AR/EN) semantic reorder of evidence. Fails safe to prior order. `COHERE_API_KEY`.
- **Pinecone RAG** (`src/lib/ai/rag.ts`) — hybrid dense+BM25 retrieval, index `cognitive-fortress`, namespaces `deepreal-cases|mhgap-modules|religion-tafsir`. `PINECONE_API_KEY/HOST`.
- **NVIDIA Content Safety** (`src/lib/safety/nvidia-content-safety.ts`) — Nemotron-3.5 Content Safety 4B moderation (allow/rewrite/block/flag).
- **Output Enforcer / One-Law** (`src/lib/ai/output-enforcer.ts`) — downgrades any AI verdict to UNVERIFIED unless a real whitelisted Tier S–C source backs it.
- **COVO Router** (`src/lib/orchestration/covo-router.ts`) — heuristic Emotional-Intent-Score (fear/guilt/superiority lexicons) + fallacy routing, intercepts queries before the LLM.
- **EAL Standard** (`src/lib/standard/*`) — executable constitution: source tier classifier, 8-layer taxonomy, logic-layer prompts, system prompts.

**Env keys observed:** `NVIDIA_API_KEY(_2..5)`, `GEMINI_API_KEY(_2..5)`, `GROQ_API_KEY(_2..4)`, `OPENROUTER/CEREBRAS/TOGETHER/SAMBANOVA_API_KEY`, `COHERE_API_KEY`, `PINECONE_API_KEY/HOST`, `GITHUB_TOKEN`, `HF_TOKEN`/`HUGGINGFACE_API_KEY`, `OPENALEX_API_KEY`, `GOOGLE_FACTCHECK_API_KEY`, `CLAIMBUSTER_API_KEY`, `SERPAPI_API_KEY`, `SIGHTENGINE_API_USER/SECRET`, `SUNNAH_API_KEY`+`SUNNAH_HADITH_SEARCH_URL`, `HADITH_API_KEY`, `HADITHAPI_COM_KEY`, `KALIMAT_API_URL/KEY`, `VERACITY_BACKEND_URL/TOKEN`, `FORENSIC_BACKEND_URL`, `EAL_SIGNING_SECRET`.

## TOP TOOLS TO INJECT INTO EVERY CHATBOT

Ranked by leverage: how much each one raises answer quality, trust, and safety per unit of integration effort. These should be wired into the shared chatbot tool layer so *any* EAL bot can call them.

1. **`/api/search/evidence` — Evidence Aggregator (7 academic APIs + Cohere rerank).** The single highest-value tool. Grounds any claim in real, free-first, trust-scored, reranked sources. Directly satisfies the One-Law ("no claim without a resolvable source"). Every chatbot answer about facts/science/health should call this first.

2. **`nvidia-first.ts` helper + Mega-Rotator — the resilient AI brain.** Not a route but the substrate: NVIDIA Nemotron-550B primary with multi-provider, multi-key failover. Inject so every bot inherits automatic rotation instead of a single fragile key.

3. **`/api/god-system` — 8-Layer analysis.** The most complete single-call analysis (safety → emotional intent → triangulation → fallacy → truth sandwich → Socratic → forward-defense + verdict). Best "do everything to this claim" tool for a general chatbot.

4. **Output Enforcer / One-Law (`src/lib/ai/output-enforcer.ts`).** Mandatory guardrail: downgrades unsourced verdicts to UNVERIFIED. Inject into every bot's response pipeline so no chatbot can present a fabricated verdict as fact.

5. **COVO Router (`covo-router.ts`) + NVIDIA Content Safety.** Pre-screen layer: emotional-manipulation scoring + AR/multilingual moderation before and after the LLM. Cheap, deterministic-ish, protects vulnerable users and routes high-emotion queries to CalmReveal.

6. **`/api/islamic/hadith` + `/api/defense/hadith-check`.** For an Egyptian audience, religious-claim grounding is as load-bearing as scientific grounding. Real graded-hadith retrieval plus isnad-grade verification stops fabricated-fatwa/hadith manipulation cold.

7. **`/api/whatsapp-analyzer`.** The native input format for Egyptian misinformation. Bot pattern + emotional-framing detection with a ready-to-paste rebuttal — the highest "shareable defense" payoff per call.

8. **`/api/forensic/image` (+ DeepReal / C2PA).** Image is the dominant viral-deception medium. EXIF + JPEG forensics + AI vision + provenance gives any chatbot a media-verification arm.

9. **`/api/search/factcheck` (Google Fact Check / IFCN).** Fast, authoritative "has this already been debunked?" lookup; ideal early-exit before spending tokens on full analysis.

10. **`/api/crisis` (verified hotlines, fail-open).** Safety net: any bot that detects distress must surface verified Egyptian hotlines first, with no analytics leak. Non-negotiable for a mental-health-adjacent platform.

**Honorable mentions:** `/api/nlp/arabic` (dialect manipulation + crisis flag header — great passive signal on every Arabic message), `/api/paper-auditor` (when a user pastes a study), and the **Cohere reranker** as a reusable relevance pass over any retrieved set.
